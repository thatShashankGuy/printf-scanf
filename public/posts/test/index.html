Working with Http Request - http.ResponseWriter , *http.Request

To demonstrate how to read request and write response. We will adding another endpoint. This time a post end point and update our front end to add a like counter. The like counter increment with a request with like and decrements if dislike button is hit.

http.HandleFunc("/like", trackLikesHandler)

We have for now print the values to the terminal but later will track this in a database when we work with database and ORM packages.

Here is the backend handler called trackLikeHandler and HTML buttons making API calls to server.

func trackLikesHandler(w http.ResponseWriter, r *http.Request) {
	if r.Method == http.MethodPost {
		err := r.ParseForm()
		if err != nil {
			http.Error(w, "Invalid form data", http.StatusBadRequest)
			return
		}

		likeCount, _ := strconv.Atoi(r.FormValue("likeCount"))
		dislikeCount, _ := strconv.Atoi(r.FormValue("dislikeCount"))
		fileName := r.FormValue("fileName")

		if likeCount > 0 {
			likeCounter += 1
		}
		if dislikeCount > 0 && likeCounter > 0 {
			likeCounter -= 1
		}

		fmt.Printf("Total likes for %s : %d \n", fileName, likeCounter)
		fmt.Fprintf(w, "Updated like counter: %d", likeCounter)
	} else {
		http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
	}
}

Focusing on HandleFunc handler we provide two arguments w http.responseWriter - Used to construct and send the HTTP response back to the client. r *http.Request - represents the incoming HTTP request, containing details such as method, URL, headers, and body.

We check API method with r.Method . The value for r.Method in this case is http.MethodPost. Based on request type it can be MethodPatch MethodPut, MethodGet and so on. This helps us avoiding defining redundent functions for same endpoint with multiple HTTP methods.

Notice that r *http.Request is a pointer to an http.Request struct. The pointer reference (*http.Request) allows the handler function to directly access and modify the data within the http.Request object, rather than working with a copy. This is efficient, especially for large requests, as it avoids duplicating data. We already have discussed Pointers in an earlier article

We get the request as form encoded data which we extract via strconv.Atoi package which a string to an integer, returning the integer value and an error if the conversion fails.

After updating our counter we send the details back in response using w. w is our responseWriter hence if the method is not specified or we need to send some error back we will pass it through w. In our example we have reponsed with an http error of method not found if the http method is not post

http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
